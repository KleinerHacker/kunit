# 温度 — 概要

パッケージ: `org.pcsoft.framework.kunit.temperature`

温度は**2つの関連グループ**でモデル化されます。温度の読み値と温度の*変化*は物理的に異なる種類の量だからです。
この区別を正しく扱うことが、演算を正しくする鍵です。

| 概念 | 型 | 性質 | 基本単位 |
|---|---|---|---|
| [絶対温度](temperature.md) | `KTemperatureUnitInstance` | アフィンな**点** | ケルビン(`K`) |
| [温度差](temperature-difference.md) | `KTemperatureDifferenceUnitInstance` | 線形の**区間** | ケルビン(`ΔK`) |

## 点 対 区間

**絶対温度**(`25 °C`、`300 K`)は温度スケール上のアフィンな*点*です。固定された零点から測定され、零点の選び方
(0 K か 0 °C か)で数値が変わります。**温度差**(`20 ΔK`)は*ベクトル* — 2点間の隔たりです。零点の負担がなく、
`20 ΔK` の差はスケール上のどこにあっても `20 °C` の差に等しくなります。

これはまさにアフィン空間の点/ベクトルの区別であり、演算を規定します:

```kotlin
import org.pcsoft.framework.kunit.of
import org.pcsoft.framework.kunit.temperature.*

val a = 30 of celsius
val b = 10 of celsius

a - b                                    // KTemperatureDifferenceUnitInstance: 20 ΔK  (区間)
a + KTemperatureDifference.ofKelvin(5)   // KTemperatureUnitInstance: 308.15 K         (依然として点)
// a + b                                 // コンパイルされません — 2つの点の加算は無意味
```

| 演算 | 結果 |
|---|---|
| `AbsTemp − AbsTemp` | **温度差**(ケルビン区間) |
| `AbsTemp + 差` | 絶対温度 |
| `AbsTemp − 差` | 絶対温度 |
| `差 ± 差` | 温度差 |
| `AbsTemp + AbsTemp` | **コンパイルエラー**(物理的に無意味) |

## 物理的に重要な理由

数式が絶対温度と差のどちらを必要とするかは、温度が**どのように**現れるかによります:

* **差 / 変化として** → **温度差**を使います。零点が打ち消されるため、ここでは `°C` と `K` は交換可能です。
  典型例 — 熱エネルギー:

  $$Q = m \cdot c \cdot \Delta T$$

  `30 °C − 10 °C = 20 ΔK` であり、`Q = m·c·20 ΔK`。熱膨張(`ΔL = α·L·ΔT`)、熱伝導、比熱・熱伝導率の単位
  (`J/(kg·ΔK)`、`W/(m·ΔK)`)も同様です。

* **乗算的に**(T 単独、べき乗 `T⁴`、比 `T₁/T₂`)→ **絶対温度(ケルビン)**を使います。絶対零度が物理の一部だから
  です:理想気体の法則 `pV = nRT`、シュテファン・ボルツマン放射 `P = εσA·T⁴`、カルノー効率 `η = 1 − T_c/T_h`。

!!! warning "同じ次元、異なる量"
    どちらのグループも次元は*ケルビン*なので、純粋な単位レベルでは `m·K` と `m·ΔK` は似て見えます。しかしここでは
    **同じ単位ではありません**:2つは異なる単位グループを使うため、絶対ケルビンを含む混合単位は差ケルビンを含む
    ものと等価でも加算可能でもありません。区別された **`ΔK`** 記号(対 `K`)は、まさにそれを `toString` や混合単位の
    出力で見えるようにするために表示されます — 独自の混合単位を組み立てる際は留意してください。
